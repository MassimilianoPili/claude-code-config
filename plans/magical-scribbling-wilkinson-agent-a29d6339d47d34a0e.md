## Argomenti Tecnici Specifici

### Loading Tiled JSON Maps in Phaser 3

**Procedura**:
1. `this.load.tilemapTiledJSON('map', 'assets/map.json')` in preload()
2. `this.load.image('tiles', 'assets/tileset.png')` per il tileset
3. `const map = this.make.tilemap({ key: 'map' })`
4. `const tileset = map.addTilesetImage('tilesetName', 'tiles')` — primo param = nome in Tiled, secondo = cache key
5. `const layer = map.createLayer('LayerName', tileset, 0, 0)`
6. `layer.setCollisionByProperty({ collides: true })` per collisioni

**NOTA**: Da Phaser 3.50+, `createStaticLayer` e `createDynamicLayer` sono unificati in `createLayer`.

**Importante per export Tiled**:
- Formato layer: CSV o Base64 (non compresso)
- Tileset: deve essere "Embedded in Map"
- Export come JSON

**Riferimenti**:
- https://blog.ourcade.co/posts/2020/phaser-3-noob-guide-loading-tiled-tilemaps/
- https://phaser.io/examples/v3.85.0/loader/tile-maps/view/load-tile-map-json
- https://docs.phaser.io/api-documentation/class/tilemaps-tilemap

**Opzione A: Free Movement (Arcade Physics)**
- `this.physics.add.sprite(x, y, 'player')`

### Scene Transitions con Fade

```javascript
// Fade out dalla scena corrente
this.cameras.main.fadeOut(1000, 0, 0, 0); // ms, r, g, b
this.cameras.main.once(Phaser.Cameras.Scene2D.Events.FADE_OUT_COMPLETE, () => {
    this.scene.start('NextScene');
});

// Fade in nella nuova scena (nel create() della scena target)
this.cameras.main.fadeIn(1000, 0, 0, 0);
```

### NPC Interaction (Proximity + Action Key)

**Pattern consigliato**:
1. Creare una Zone/overlap area attorno all'NPC
2. `this.physics.add.overlap(player, npcZone, callback)` per rilevare prossimita'
3. Mostrare indicatore "Premi E" quando il player e' nella zona
4. Listener per tasto azione: `this.input.keyboard.on('keydown-E', ...)`
5. Se overlap attivo + tasto premuto → apri dialogo

**Esempio overlap zone**:
- https://phaser.io/examples/v3.85.0/physics/arcade/view/overlap-zone
- https://phaser.io/examples/v3.85.0/physics/arcade/view/overlap-event

**ATTENZIONE**: `body.touching` funziona solo se il player si muove (richiede velocity). Per NPC fermi, usare `overlap` invece di `collider`.

### Camera Follow + Zoom per Pixel Art

```javascript
// Config Phaser
const config = {
    type: Phaser.AUTO,
    pixelArt: true,          // CRITICO: disabilita antialiasing
    scale: {
        mode: Phaser.Scale.FIT,
        autoCenter: Phaser.Scale.CENTER_BOTH,
        width: 320,           // Risoluzione "virtuale" bassa per pixel art
        height: 240
    },
    // ...
};

// Camera follow nel create()
this.cameras.main.startFollow(player, true);  // true = roundPixels
this.cameras.main.setZoom(2);                 // DEVE essere intero per pixel art
// Oppure con lerp per smoothing:
this.cameras.main.startFollow(player, true, 0.1, 0.1); // lerp x, lerp y
```

**Regole per pixel art**:
- `pixelArt: true` nella config (equivale a `render.antialias: false` + `render.pixelArt: true`)
- Zoom SEMPRE intero (1, 2, 3...) — mai 1.5 o 2.5
- `roundPixels: true` nel camera follow
- Risoluzione virtuale bassa (es. 320x240 o 256x224) + Scale.FIT

**Riferimenti**:
- https://docs.phaser.io/phaser/concepts/cameras
- https://phaser.discourse.group/t/help-with-scaling-for-pixel-art/4782
